Restrictions - shorthand version

Restrictions has two overloaded forms - one that takes a single group of roles, and another that takes multiple groups. A group is represented by an array.

The first form of Restrictions uses an ANDed set of roles to determine access to an action. For example, a user with the roles "foo" and "bar" could access a Restrictions-protected action that specified any of the following:

  • foo
  • bar
  • foo AND bar

However, a Restrictions that required "foo", "bar" and "hurdy" would deny access to the user.

Specification
Result
Restrictions(Array("foo"), new MyDeadboltHandler) {
  Action {
    Ok(accessOk())
  }
}
Restrictions(Array("foo", "bar"), new MyDeadboltHandler) {
  Action {
    Ok(accessOk())
  }
}
Restrictions(Array("foo", "!bar"), new MyDeadboltHandler) {
  Action {
    Ok(accessOk())
  }
}
Restrictions(Array("hurdy"), new MyDeadboltHandler) {
  Action {
    Ok(accessOk())
  }
}
